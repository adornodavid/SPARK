<context>
El problema raíz es que en Vercel serverless, las variables de entorno pueden no estar disponibles cuando se evalúa el módulo (top-level). El patrón `const supabase = createClient(url, key)` a nivel de módulo funciona en dev (Node.js persistente) pero puede fallar en serverless (cold start con env vars tardías). La solución fue mover createClient dentro de las funciones específicas que fallaban. Si el fix funciona, idealmente se debería aplicar el mismo patrón a todas las server actions, pero por ahora solo se corrigieron las dos reportadas.
</context>
